**Подводные камни / lessons learned:**

1. **Worktree desync.** Я работал в `.claude/worktrees/admiring-faraday-cec21c/`,
   юзер — в основном проекте `/Users/ustozferuz/Kruger/`. Файлы между ними
   расходились (10+ компонентов, новые products/Pagination, новая WorldMap
   с @svg-maps/world). Всегда сверяться: `diff -rq /Users/ustozferuz/Kruger/src src/`
   если юзер жалуется что не видит изменений.

2. **Tailwind 4 + кастомные `--z-*` токены.** `--z-header: 65` в `@theme inline`
   не всегда генерирует `.z-header` класс даже после рестарта dev (vite cache /
   content scanning quirk). Workaround: `z-[65]` arbitrary value.

3. **`bg-bg/95` + `backdrop-blur-sm` на Header → багу с просвечиванием карточек.**
   Декоративные полоски `z-10` внутри ProductCard видны через прозрачный header
   при скролле. Фикс: solid `bg-bg` + JS-тень при скролле через class toggle.

4. **TopBar не sticky → LanguageSwitcher dropdown под Header.** TopBar в обычном
   потоке, его дропдаун (z-50) ниже Header (z-65). Поднять до z-[70].

5. **Hover-pattern dropdown vs Radix Popover.** «О компании» использует
   `group-hover:block` (без Portal), не Radix DropdownMenu. Это ОК если:
   - z-index у дропдауна > z-index Header'а (или header не имеет stacking context)
   - Header не имеет `overflow: hidden`

6. **Ангрить-фактор юзера.** Когда я перестраивал контент (контактную страницу,
   убирал DE из regional list — он считал это «портить», т.к. сам активно делал
   там работу). Правило: если юзер активно над чем-то работает — делать
   точечные минимальные правки, не «улучшать общую структуру».
